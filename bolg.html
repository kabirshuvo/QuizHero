Q:1 Can you tell me how .map(), .filter(), .forEach(), and .find() are different in JavaScript?

  • .map(): By using .map() function we can easily a create a new array from the original array or array of object. .map() function always returns something and we can easily store that returns in a variable.
  • .filter(): By using a condition with .filter() function we can confidently create an array, with the desired retruns form the original array or array of object.
  • .forEach(): To iterate and to apply a function over all elements of an array. .forEach() function dont returns anything.
  • .find(): To find out a specific and single element form an array. Basically it returns the first one.

In summary, .map() and .filter() both creates a new array, whereas .forEach() and .find() do not.
.forEach() applies a function to each element without modifying the array, while .map() and .filter() both modify the array or create a new one.
Finally, .filter() returns an array of elements that pass a certain condition, while .find() returns only the first element that passes the condition.

Q:2 What is the differences between LocalStorage, Session Storage and cookies?

localStorage, sessionStorage, and cookies allows us to store data on the client-side of a web browser.

localStorage: With no expiration date localStorage allows us to store key-value pairs in a web browser's storage. This data will persist even after the browser is closed, until explicitly cleared.

sessionStorage: it is as similar as localStorage, but the only difference is sessionStorage stored data is cleared when the user closes the browser tab or window or with the session ends.

Cookies: Cookies are primarily used for tracking user activity or for session management, and have a limited storage capacity of up to 4KB.
